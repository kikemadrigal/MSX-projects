20200525 Release
================
- DRC does now support labels for SKIP jumps just like DC did. A label is a $ sign followed by some text (i.e. $loop). You can put a label just before an entry, and the use SKIP <label> (i.e. "SKIP $loop") to jump back to that entry. Labels in between condacts are not allowed. You can add more than one label before an entry, so you can jump there using any of them as reference. Forward jumps are allowed.
- Some error texts have been improved to send a clearer message.
- Now if you call a process that does not exist, DRC will detect it in compile time.

20191220 Release
================

- DRC now supports XSPLITSCR fake condact for Amstrad CPC. This allows splitting the screen in two parts with two different videomodes from the three Amstrad CPC standard ones.
- DRF now supports XBEEP for CPC and MSX via Maluva. C64 and ZX natively support BEEP and XBEEP just replaces BEEP for machines not supporting it. Parameters for BEEP are duration (in 1/50s of second) and tone (a value from 48 to 238)
- DRF now supports XPLAY condact, which accepts an MML string at the MSX or Spectrum +3 style for basic PLAY, but just for one channel. DRC will actually convert that into BEEPs or XBEEPs depending on target.
- DRF now supports identifiers in list entries, that is, you can use /xxxxxx, where xxxxxx is an indetifier previously defined with #define wherever you used /1 /2 /3 /4... /n before (/OBJ, /OTX, /MTX, /STX, /LTX, and /CON sections)
- DRF now makes some semantic analysis, such as detecting if some condact which requires a locno, objno, mesno, etc.  parameter is refering to a location, object, message that does actually exist
- DRF now supports a few parameters. Any parameter after the DSF or JSON file names, are candidates to be options, or a symbol list. If starting by a hyphen '-' they will be considerec a parameter, otherwise, a comma-separated symbol list 
  New parameter options:
  -verbose :  verbose output
  -no-semantic : does not make semantic analysis (so DRF works as in previous versions)
  -semantic-warnings : when semantic analysys finds something wrong, just raises a warning instead of stopping with an errors
  -force-normal-messages : no matter the target, all XMESSAGES will be treated as normal messages

  Examples:

  drf test.dsf -verbose symbol1,symbol2
  drf test.dsf test.json symbol1,symbol2 -force-normal-messages
- DRC now has a wiki in Github
- Added support for Maluva Amstrad PCW

20191105 Release
================

- DRC now supports new fake condacts XMESSAGE, XMES, XPICTURE, XSAVE and XLOAD. These condacts are related with Maluva Extension, and will make it easier for authors to handle Maluva features.
  With this extension you can load additional messages from disk, raster images, save/load game to disk, etc, and these condacts will handle that. See Maluva documentation for more info.
- DRC will now determine which Maluva binary to use automatically based on target. Thus you can just add 

  #extern "MALUVA"

  to your code, and DRC will replace that MALUVA with MLV_MSX.BIN, MLV_CPC.BIN, MLV_C64.BIN, etc. depending on target. In order to make this possiblem, DRF requires now the author to specify absolutely
  subtarget for ZX target. Options are ESXDOS, PLUS3 and NEXT. If you are not using Maluva, you must specify a subtarget to, choose any subtarget indistinctly.

20190518 Release
================

- DRB is now less verbose but it has a new -v parameter which increases verbosity
